## 2. 後台登入與登出

### 2.1 登入

1. 開啟 `https://impeanut.com/admin/login.html`
2. 輸入管理員 Email 與密碼
3. 按下登入
4. 登入成功後進入後台

### 2.2 登出

1. 在後台頁面按「登出」
2. 系統會清除登入狀態
3. 返回登入頁

### 2.3 登入失敗排查

- 確認 Email 與密碼是否正確
- 確認 Supabase Authentication 使用者仍存在
- 確認網路正常
- 開啟瀏覽器 Console 查看錯誤
- 若頁面持續跳回登入頁，檢查登入 Session 是否過期

## 3. 訂單管理

### 3.1 訂單處理建議流程

1. 新訂單進入
2. 確認付款方式與付款狀態
3. 確認訂單內容
4. 準備商品
5. 完成包裝
6. 完成出貨
7. 訂單完成

### 3.2 訂單狀態

常用狀態包含：

- 新訂單
- 已確認
- 處理中
- 已完成
- 已取消

### 3.3 付款狀態

- 待付款
- 已付款
- 付款失敗
- 已退款

### 3.4 包裝狀態

- 未準備
- 備貨中
- 已備貨
- 已包裝

### 3.5 出貨狀態

- 未出貨
- 待出貨
- 已出貨
- 已送達

### 3.6 取消訂單

建議將測試訂單或取消訂單改成「已取消」，不要直接刪除。

原因：

- 保留操作紀錄
- 避免破壞訂單與商品明細關聯
- 品牌分析會排除已取消訂單的營收
- 日後可追查顧客問題

### 3.7 操作紀錄

訂單詳情頁底部可查看操作紀錄，包含狀態變更時間與操作內容。

## 4. 商品管理

### 4.1 新增商品

1. 進入商品管理
2. 按新增商品
3. 填寫商品名稱、SKU、價格、分類與說明
4. 上傳商品圖片
5. 設定是否顯示與上架狀態
6. 儲存

### 4.2 商品狀態

- `active`：上架
- `inactive`：下架
- `is_visible = true`：前台顯示
- `is_visible = false`：前台隱藏

### 4.3 修改商品

修改價格、內容或圖片後，先在前台確認商品詳情頁是否正常，再推送正式版本。

### 4.4 商品圖片建議

- 使用 WebP 或壓縮 JPG
- 避免單張圖片過大
- 保持一致尺寸與比例
- 檔名使用英文、小寫與連字號

## 6. 文章管理

### 6.1 文章分類

- 花生研究室
- 最新消息
- 品牌故事
- 檢驗報告

### 6.2 發布流程

1. 建立文章
2. 填寫標題、摘要與內文
3. 設定分類
4. 填寫 SEO 標題與描述

- 標題清楚描述文章主題
- 摘要自然包含主要關鍵字
- 不重複堆疊關鍵字
- 每篇文章使用獨立標題與描述

## 11. Google 試算表訂單同步

### 11.1 目前狀態

- 手動同步：已完成
- 自動同步：尚未開啟
- 測試工作表：`Orders_Test`

### 11.2 手動同步

在 Apps Script 執行：

```text
exportOrdersToSheet
```

### 11.3 同步邏輯

- 新訂單：新增一列
- 舊訂單：依資料庫 ID 更新原本資料
- 不會重複新增
- 會同步商品明細、金額與各種狀態

### 11.4 開啟自動同步

正式上線前，到 Apps Script：

1. 點左側「觸發條件」
2. 新增觸發條件
3. 函式選 `exportOrdersToSheet`
4. 事件來源選「時間驅動」
5. 選「日計時器」
6. 建議時段：凌晨 1 點至 2 點
7. 儲存

### 11.5 關閉自動同步

到觸發條件頁，找到 `exportOrdersToSheet`，按右側三點並刪除觸發條件。

### 11.6 金鑰安全

`service_role` 金鑰只能存放在 Apps Script 指令碼屬性，不得：

- 貼到聊天室
- 放進 GitHub
- 放進前台 JavaScript
- 公開截圖

## 12. GitHub 更新流程

每次修改完成後，在 Terminal 執行：

```bash
git add .
git commit -m "更新說明"
git push
```

推送後等待 GitHub Pages 部署，再開正式網址測試。

### 12.1 建議提交訊息

- `feat:` 新功能
- `fix:` 修正問題
- `docs:` 文件更新
- `style:` 版面調整
- `refactor:` 程式重整

## 14. 常見問題排除

### 14.1 網站修改後沒有更新

- 確認是否已 `git push`
- 等待 GitHub Pages 部署
- 使用 `Cmd + Shift + R` 強制重新整理
- 檢查檔案版本號

### 14.2 GA4 沒有資料

- 確認 GA4 ID 已存入系統設定
- 檢查 `analytics.js` 是否上線
- 開啟 Console 查看是否顯示「GA4 已啟用」
- 到 GA4 即時報表測試

### 14.3 試算表沒有訂單

- 確認 Apps Script 指令碼屬性
- 確認使用正確 Supabase URL
- 確認使用 `service_role`，不是 `anon`
- 查看 Apps Script 執行紀錄

### 14.4 後台讀不到資料

- 確認已登入
- 檢查 Console
- 確認 Supabase RLS 與 Policy
- 確認資料表欄位名稱
